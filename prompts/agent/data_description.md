The data source contains two POA (Point of Acquisition) Sales fact tables used by the sales team. fact_sales_billing stores realized sales (billing) by month, customer, product groups, regions, brands, and sales reps. fact_sales_booking stores order bookings before shipment using similar business dimensions. Both tables include metrics such as total_sales and quantities, with aligned fields like year_month, order_type, brand, ru, sub_unit, customer_parent, g7, and flags for OEM/EMS/G7. Use billing for actual revenue; use booking for demand and pipeline visibility.
